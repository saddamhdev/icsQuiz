# üß† **Flow Explanation (Very Clear)**

### üëâ **1. User hits login API**

User ‚Üí Ingress ‚Üí Service ‚Üí WebFlux Pod

### üëâ **2. WebFlux Pod does:**

1. Check Redis (fast, single lookup)
2. If not found ‚Üí Query PostgreSQL
3. After DB hit ‚Üí store user in Redis
4. Kafka used for async events:

   * Login logs
   * Auth activity
   * User analytics

---

# ‚ö° **3. Redis**

Purpose:

* Store user login details (cached)
* Reduce DB load
* Increase speed √ó10

Latency:

```
~1 ms lookup
```

---

# ‚ö° **4. PostgreSQL**

Purpose:

* Main database
* Stores student table, credentials, staging user

Query latency (with index):

```
4‚Äì8 ms average
```

---

# ‚ö° **5. Kafka (KRaft)**

Used for:

* Logging login events
* Analytics system
* Notification events
* Future exam real-time scoring

Kafka decouples load ‚Üí system never slows.


```
1 pod ‚Üí 4 pods automatically
```
